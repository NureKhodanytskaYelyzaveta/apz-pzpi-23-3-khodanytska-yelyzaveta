2 ЗАВДАННЯ 
Метою практичного заняття є ознайомитися з одним із шаблонів (патернів) проєктування програмного забезпечення, проаналізувати його структуру, призначення та практичне застосування.

3 ОПИС ВИКОНАНОЇ РОБОТИ 
3.1 Загальні відомості про шаблони (патерни) проєктування
Шаблони (патерни) проєктування — це типові, перевірені рішення поширених проблем, які виникають у процесі розробки програмного забезпечення. Вони не є готовими фрагментами коду, а описують підхід до побудови структури програми та взаємодії її компонентів.
Основне призначення патернів полягає у спрощенні процесу розробки, підвищенні якості коду, забезпеченні його гнучкості, розширюваності та повторного використання. Використання патернів дозволяє створювати більш зрозумілі та підтримувані програмні системи.
Згідно з класифікацією GoF, шаблони проєктування поділяються на три основні групи:
породжувальні — відповідають за створення об’єктів;
структурні — визначають способи побудови класів та об’єктів;
поведінкові — описують взаємодію між об’єктами та розподіл відповідальностей.
Патерни застосовуються в об’єктно-орієнтованому програмуванні для побудови масштабованих, гнучких і легко модифікованих систем.
3.2 Опис обраного патерна
Назва патерна: Strategy.
Тип шаблона (патерна): поведінковий.
Основне призначення:
Патерн Strategy призначений для визначення сімейства алгоритмів, інкапсуляції кожного з них в окремий клас та забезпечення їх взаємозамінності. Це дозволяє змінювати алгоритм незалежно від клієнта, який його використовує.
Застосування патерна є доцільним у випадках, коли:
існує декілька способів виконання однієї задачі;
необхідно уникнути великої кількості умовних операторів (if/else, switch);
потрібно забезпечити можливість зміни поведінки об’єкта під час виконання програми;
важливо ізолювати алгоритми від основної логіки програми.
Переваги:
дотримання принципу Open/Closed (можливість розширення без зміни існуючого коду);
ізоляція алгоритмів у окремих класах;
спрощення тестування;
використання композиції замість наслідування;
підвищення гнучкості системи.
Недоліки:
збільшення кількості класів у програмі;
необхідність вибору конкретної стратегії клієнтом;
можливе ускладнення структури для простих задач.
3.3 Структура шаблона (патерна)
Патерн Strategy складається з декількох основних структурних елементів, кожен з яких виконує окрему функцію в системі.
Опис структурних частин:
Strategy — інтерфейс, який визначає спільний метод для всіх алгоритмів;
ConcreteStrategy — конкретні класи, що реалізують різні варіанти алгоритмів;
Context — клас, який використовує обрану стратегію через інтерфейс Strategy.
Клієнт створює об’єкт контексту та передає йому обрану стратегію. Контекст зберігає посилання на інтерфейс Strategy і викликає відповідний метод, не знаючи деталей реалізації. Конкретна стратегія виконує необхідний алгоритм і повертає результат.
Таким чином забезпечується слабке зв’язування між класами та можливість динамічної зміни поведінки програми.
На UML-діаграмі показано взаємозв’язок між інтерфейсом Strategy, ConcreteStrategy та класом Context. Контекст використовує стратегію через інтерфейс, що дозволяє підставляти різні реалізації без зміни його коду.
3.4 Реалізація шаблона (патерна)
Реалізація патерна Strategy була виконана мовою програмування C#, було створено інтерфейс для визначення спільної поведінки, а також кілька класів, що реалізують різні варіанти алгоритмів.
Основні етапи:
1. Створення базового інтерфейсу IPaymentStrategy, який містить метод Pay(). Даний інтерфейс визначає контракт для всіх стратегій.
2. Створено класи CardPayment, CashPayment, PayPalPayment, які реалізують інтерфейс IPaymentStrategy. Кожен із цих класів містить власну реалізацію методу Pay().
3. Клас ShoppingCart виступає в ролі контексту, який зберігає посилання на інтерфейс стратегії та викликає метод оплати через нього.
4. У програмі реалізовано приклад, де користувач може обирати спосіб оплати. В залежності від вибору встановлюється відповідна стратегія, після чого викликається метод Checkout(). Це демонструє можливість використання різних алгоритмів без зміни основної логіки програми.

Програмний код 
	Напиши на C# програму про оплату покупок через різні способи. Зроби інтерфейс оплати з методом Pay. Реалізуй три варіанти: картка, готівка і PayPal. Створи кошик, де можна вибрати спосіб оплати і зробити покупку. Якщо спосіб не вибрали, покажи помилку. У main покажи три покупки з різними способами оплати.
В.1 Використаний у презентації код
 1 // Інтерфейс, який визначає контракт для всіх стратегій оплати
 2 public interface IPaymentStrategy
 3 {
 4     void Pay(int amount);
 5 }
 6 
 7 // Стратегія: Оплата кредитною карткою
 8 public class CreditCardPayment : IPaymentStrategy
 9 {
10     private string _cardNumber;
11 
12     public CreditCardPayment(string cardNumber)
13     {
14         _cardNumber = cardNumber;
15     }
16 
17     public void Pay(int amount)
18     {
19         Console.WriteLine($"Оплата карткою {_cardNumber} на суму {amount} грн");
20     }
21 }
22 
23 // Стратегія: Оплата готівкою
24 public class CashPayment : IPaymentStrategy
25 {
26     public void Pay(int amount)
27     {
28         Console.WriteLine($"Оплата готівкою на суму {amount} грн");
29     }
30 }
31 
32 public class ShoppingCart
33 {
34     private IPaymentStrategy _paymentStrategy;
35 
36     // Метод для встановлення стратегії оплати
37     public void SetPaymentStrategy(IPaymentStrategy strategy)
38     {
39         _paymentStrategy = strategy;
40     }
41 
42     // Метод оформлення замовлення
43     public void Checkout(int amount)
44     {
45         if (_paymentStrategy == null)
46         {
47             Console.WriteLine("Помилка: стратегія оплати не обрана");
48             return;
49         }
50 
51         Console.WriteLine($"Оформлення замовлення на суму {amount} грн");
52         _paymentStrategy.Pay(amount);
53         Console.WriteLine("Замовлення успішно оформлено");
54     }
55 }
56 
57 // Нова стратегія: Оплата через PayPal
58 public class PayPalPayment : IPaymentStrategy
59 {
60     private string _email;
61 
62     public PayPalPayment(string email)
63     {
64         _email = email;
65     }
66 
67     public void Pay(int amount)
68     {
69         Console.WriteLine($"Оплата через PayPal ({_email}) на суму {amount} грн");
70     }
71 }
72 
73 class Program
74 {
75     static void Main(string[] args)
76     {
77         Console.OutputEncoding = System.Text.Encoding.UTF8;
78         var cart = new ShoppingCart();
79 
80         // Сценарій 1: Оплата кредитною карткою
81         Console.WriteLine("Покупка 1:");
82         cart.SetPaymentStrategy(new CreditCardPayment("****1234"));
83         cart.Checkout(1500);
84 
85         Console.WriteLine();
86 
87         // Сценарій 2: Оплата готівкою
88         Console.WriteLine("Покупка 2:");
89         cart.SetPaymentStrategy(new CashPayment());
90         cart.Checkout(300);
91 
92         Console.WriteLine();
93 
94         // Сценарій 3: Оплата PayPal
95         Console.WriteLine("Покупка 3:");
96         cart.SetPaymentStrategy(new PayPalPayment("user@example.com"));

98     }
