ВИКОРИСТАННЯ ШІ ПІД ЧАС РОЗРОБКИ
Назва інструменту та версія: модель GPT-5.3, план Go
Мета використання: Використання ШІ було необхідне через відсутність досвіду роботи з Docker, Kubernetes та процесами масштабування бекенд-систем. ШІ допомагав зрозуміти принципи контейнеризації, роботу Kubernetes (Deployment, Service), налаштування балансування навантаження та організацію взаємодії між компонентами системи.
Внесок автора: Я самостійно створила Docker-образ бекенд-застосунку, налаштувала docker-compose для локального запуску, написала конфігурації Kubernetes (Deployment та Service), розгорнула систему в Minikube, виконала масштабування через зміну кількості подів, провела навантажувальне тестування та проаналізувала результати. Також я виправляла помилки під час деплою, налаштовувала мережеву взаємодію та адаптувала конфігурації під власний проєкт.

1. Запит для створення Docker-контейнера
Запит до ШІ:
як створити docker контейнер для flask бекенду і що має бути в dockerfile
Результат:
ШІ пояснив структуру Dockerfile, процес збірки образу та запуск контейнера. Я створила власний Docker-образ для бекенду та інтегрувала його у систему.
Фрагмент коду, отриманий від ШІ:
FROM python:3.10
WORKDIR /app
COPY . .
RUN pip install -r requirements.txt
CMD ["python", "app.py"]

2. Запит для docker-compose та зв’язку сервісів
Запит до ШІ:
як з'єднати flask і nginx через docker-compose
Результат:
ШІ показав, як описати кілька сервісів і налаштувати між ними зв’язок. Я реалізувала конфігурацію з Flask-бекендом і Nginx як проксі-сервером.
Фрагмент коду, отриманий від ШІ:
services:
  backend:
    build: .
  nginx:
    image: nginx
    depends_on:
      - backend

3. Запит для налаштування Kubernetes Deployment
Запит до ШІ:
як створити deployment в kubernetes для flask і задати кількість реплік
Результат:
ШІ пояснив структуру deployment.yaml та параметр replicas. Я реалізувала deployment із кількома подами для масштабування.
Фрагмент коду, отриманий від ШІ:
apiVersion: apps/v1
kind: Deployment
spec:
  replicas: 2

4. Запит для створення Service і балансування
Запит до ШІ:
як зробити load balancer в kubernetes для розподілу запитів
Результат:
ШІ показав використання Service типу LoadBalancer. Я налаштувала сервіс для рівномірного розподілу трафіку між подами.
Фрагмент коду, отриманий від ШІ:
kind: Service
spec:
  type: LoadBalancer

5. Запит для масштабування системи
Запит до ШІ:
як змінювати кількість подів у kubernetes
Результат:
ШІ пояснив використання команди kubectl scale. Я застосувала це для тестування різної кількості подів і аналізу продуктивності.
Фрагмент коду, отриманий від ШІ:
kubectl scale deployment flask-backend --replicas=5

6. Запит для пошуку та виправлення помилок
Запит до ШІ:
чому не запускається контейнер або не працює service в kubernetes
Результат:
ШІ допоміг знайти причини помилок (мережа, порти, конфігурації). Я самостійно виправила помилки та налагодила роботу системи.
Фрагмент коду, отриманий від ШІ:
kubectl logs <pod_name>
